# Tool Storage Location Quick Reference

**Last Updated:** 2026-02-01

---

## Quick Decision Tree

```
Is this a PERMANENT tool or TEMPORARY file?

PERMANENT → Continue below
TEMPORARY → claude/developer/workspace/

Is it specific to ONE agent or SHARED?

AGENT-SPECIFIC → claude/agents/<agent-name>/
SHARED → Continue below

What category?

BUILD AUTOMATION → claude/developer/scripts/build/
TEST SCRIPTS → claude/developer/scripts/testing/
ANALYSIS TOOLS → claude/developer/scripts/analysis/
AGENT UTILITIES → claude/developer/scripts/agent-tools/
MANAGER WORKFLOWS → claude/manager/scripts/
RELEASE AUTOMATION → claude/release-manager/scripts/
SECURITY TOOLS → claude/security-analyst/scripts/
OTHER → claude/developer/scripts/<category>/
```

## Storage Locations

### Agent-Specific Tools

**Location:** `claude/agents/<agent-name>/`

**When to use:**
- Tool is only useful for one specific agent
- Contains agent-specific data or references
- Tightly coupled to agent's domain

**Structure:**
```
claude/agents/<agent-name>/
├── scripts/      # Automation scripts
├── tools/        # Utility programs
├── data/         # Indexes, caches, reference data
├── templates/    # Code/document templates
└── README.md     # Tool documentation
```

**Examples:**
- `.claude/agents/msp-expert/scripts/build_message_index.py`
- `.claude/agents/aerodynamics-expert/data/quick_lookup_index.json`
- `.claude/agents/test-engineer/templates/basic_sitl_test.py`

---

### Shared Developer Scripts

**Location:** `claude/developer/scripts/<category>/`

**When to use:**
- Multiple agents use it
- Users run it directly
- General-purpose utility

**Existing categories:**
```
claude/developer/scripts/
├── build/         # Build automation (inav-builder, fc-flasher)
├── testing/       # Test scripts (test-engineer, sitl-operator)
├── analysis/      # Code analysis tools
└── agent-tools/   # Shared agent utilities (create as needed)
```

**Examples:**
- `claude/developer/scripts/testing/diagnose_sitl_ports.sh`
- `claude/developer/scripts/build/verify_hex_file.py`
- `claude/developer/scripts/agent-tools/parse_git_history.py`

---

### Role-Specific Scripts

**Locations:**
- `claude/manager/scripts/` - Project management automation
- `claude/release-manager/scripts/` - Release workflows
- `claude/security-analyst/scripts/` - Security analysis tools

**When to use:**
- Specific to role's responsibilities
- Not agent-specific
- Used across multiple sessions

**Examples:**
- `claude/manager/scripts/bulk_archive_projects.sh`
- `claude/release-manager/scripts/verify_release_assets.py`
- `claude/security-analyst/scripts/detect_crypto_misuse.py`

---

### Templates

**Locations:**
- `claude/agents/<agent-name>/templates/` - Agent-specific
- `claude/manager/templates/` - Project templates
- `claude/release-manager/templates/` - Announcement templates
- `claude/security-analyst/templates/` - Report templates

**When to use:**
- Boilerplate code/documents
- Starting point for common tasks
- Standardized formats

---

### Temporary Workspace

**Location:** `claude/developer/workspace/`

**When to use:**
- Working files for current session
- Analysis in progress
- Scratch files
- Investigation notes

**NOT for:**
- Permanent tools
- Reusable scripts
- Reference data
- Templates

**Cleanup:** Files here may be deleted between sessions

## Examples by Use Case

### Building a Reusable Test Script

**If:** Only test-engineer agent needs it
**→** `.claude/agents/test-engineer/scripts/create_msp_test.py`

**If:** Multiple agents and users need it
**→** `claude/developer/scripts/testing/inav/msp/create_msp_test.py`

### Creating a Data Index

**If:** Only msp-expert agent uses it
**→** `.claude/agents/msp-expert/data/msp_index.json`
**→** `.claude/agents/msp-expert/scripts/build_index.py` (to generate it)

**If:** Multiple agents need it
**→** `claude/developer/scripts/agent-tools/build_msp_index.py`
**→** Output to appropriate location

### Manager Automation

**If:** Project status updates
**→** `claude/manager/scripts/update_all_project_statuses.py`

**If:** Email template
**→** `claude/manager/templates/task-assignment.md`

### Temporary Analysis

**Always:** `claude/developer/workspace/current-analysis/`

## Documentation Requirements

### Agent README

Create `claude/agents/<agent-name>/README.md` for agent-specific tools:

```markdown
# <Agent Name> Tools

## Scripts

### script_name.py
**Purpose:** One-line description
**Usage:** ./.claude/agents/<agent>/scripts/script_name.py <args>
**Created:** YYYY-MM-DD - Why it was created

## Data

### index_file.json
**Purpose:** What it contains
**Generated by:** Which script creates it
**Last updated:** YYYY-MM-DD
```

### Shared Script Documentation

Use header comments in shared scripts:

```python
#!/usr/bin/env python3
"""
Brief: One-line description
Usage: ./script_name.py <args>
Example: ./script_name.py "example"

What problem this solves: Why this exists
When to use it: What situations call for this
"""
```
